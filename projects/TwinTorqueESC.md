## Project Goal

Design and implement an open-source dual-motor ESC based on the STM32G431KBT6, driven by DSHOT from a separate flight controller. The project proceeds in two parallel tracks:

1. **ESC firmware** — register-level STM32 C, no HAL
2. **ESC hardware** — KiCad schematic and PCB (later phase)

---

## Two-Board Architecture

### Flight Controller (FC)
- **Hardware:** STM32G491RE (Nucleo-64, ordered)
- **Role:** IMU, barometer, radio/GPS, outputs DSHOT signals only
- **Status:** Not started — FC development comes after ESC is working

### ESC Board
- **Hardware:** STM32G431KBT6 (32-pin LQFP, bare chips in stock, also Nucleo-32 boards available)
- **Role:** Dual motor ESC — two motors per chip, one chip per board, two boards for a quad
- **Timer assignment:** TIM1 → Motor 1, TIM8 → Motor 2
- **Status:** Active development — starting now

## Target Motor

**T-Motor F1404 2900KV** (ordered, not yet arrived)
- 3–4S LiPo
- Peak current: 20A (60s)
- Internal resistance: 138mΩ
- 9N12P stator

---

## Platform Direction

This bench build uses the F1404 as a learning and validation platform — the full firmware stack (DSHOT, trapezoidal, BEMF, FOC, telemetry) is developed and proven here. The long-term goal is an **endurance platform** (larger frame, lower KV motors, 7"+ props, Li-Ion cells, 40–60 min flight time). The power stage components and variant config system are designed to scale — the G431 firmware carries across unchanged.

## Battery (Flight)

**GNB 1000mAh 4S 110C LiHV (XT30)**
- Chemistry: LiHV — cells charge to 4.35V each (vs 4.20V standard LiPo)
- Voltage: 15.2V nominal, **17.4V fully charged**
- Weight: 108g
- Dimensions: 27×35×65mm
- Discharge: 110C continuous = 110A — well over-specced for 4× F1404 peak of 80A total
- Connector: XT30 — plan XT30 on ESC board (appropriate for this power level, saves weight vs XT60)
- Charge rate: 1C–5C

**Charger note:** Must select **LiHV mode** on charger (4.35V/cell cutoff). Charging in standard LiPo mode (4.20V/cell) will leave ~15% capacity unused.

**Bench testing:** Bench PSU (30V/5A) used for all pre-prop bring-up. Battery only connected when prop is attached for real thrust testing.

**CODDAR 2200mAh 4S 110C LiPo (XT60)** — also available, used for bench sessions where long runtime between charges is more important than weight. Too heavy (~240g) for flight with F1404 platform.

| Component | Part | Notes |
|-----------|------|-------|
| MCU | STM32G431KBT6 | 170MHz Cortex-M4, FPU, CORDIC, FMAC, internal opamps |
| Gate driver | DRV8300DPWR | TSSOP-20, 100V, 3-phase, integrated bootstrap diodes |
| Power FETs | BSC014N04LSATMA1 | Infineon OptiMOS5, 40V, 1.4mΩ Rds(on), SuperSO8 |
| GVDD buck | TPS54202DDCR | 4.5–28V in, 2A, SOT-23-6, set to 12V |
| Current sense | INA181A1IDBVR | SOT-23-6, gain 20V/V, low-side shunt |
| TVS clamp | SMAJ20A | Across power rail |
| Schottky | SS34 | Reverse polarity protection |
| Inductor | SRR1260-100M | 10µH, 5.5A, for TPS54202 (proto) |
| Inductor (PCB) | SRR0604-4R7ML | 4.7µH, 2.55A, smaller footprint for final PCB |

### DRV8300 Key Facts
- GVDD supply range: 4.8–20V (12V used here, generated by TPS54202)
- 750mA source / 1.5A sink peak gate current
- Integrated bootstrap diodes (D variant = DRGER/DPWR)
- Dead-time set by resistor on DT pin (200ns–2000ns) — TSSOP variant has fixed 200ns
- Hardware shoot-through prevention
- No SPI — purely PWM input controlled

### BSC014N04LS Key Facts
- Qg @ 4.5V = 31nC (relevant figure for DRV8300 drive level)
- Qg @ 10V = 61nC
- Ciss = 4300pF
- Rise time 9ns
- SuperSO8 5×6mm — thermal pad on bottom, reflow with hotplate

## Prototyping Approach

- Breakout boards (LCQT-SOT23-6, LCQT-TSSOP20 in workshop) for SMD parts
- Perfboard with Nucleo-32 for initial bring-up
- Bench PSU replaces battery for all pre-prop testing (safe up to ~2A unloaded)
- 470–1000µF electrolytic cap across PSU output terminals (PSU lead inductance compensation)
- KiCad PCB comes after firmware is validated

## Firmware Development Phases

### Phase 1 — 6-step trapezoidal, open loop
- TIM1 centre-aligned complementary PWM, 48kHz
- Dead-time insertion, verify on scope first
- 6-step commutation state machine
- Open-loop ramp for initial spin

### Phase 2 — Sensorless closed loop
- BEMF zero crossing detection via G431 internal comparators
- Close the commutation loop

### Phase 3 — DSHOT300 decode
- Timer input capture + DMA (not bit-banging)
- 16-bit frame: 11-bit throttle | 1-bit telemetry req | 4-bit CRC
- Arming: throttle = 0 for ~1 second before accepting non-zero
- DSHOT300 = 300kbps, frame time ~53µs

### Phase 4 — FOC
- Clarke/Park transforms
- Use G431 CORDIC hardware accelerator for sin/cos
- Use G431 FMAC for PI loop filtering

### Phase 5 — Second motor + telemetry
- TIM8 → Motor 2 (identical register map to TIM1)
- Bidirectional DSHOT (eRPM telemetry back on same wire)
- UART telemetry output

## STM32G431KBT6 Key Resources

- 170MHz Cortex-M4 with FPU
- TIM1 and TIM8 — both full advanced motor control timers, identical register maps
- 3× internal opamps — useful for current sensing (Phase 1+ can use INA181 externally)
- CORDIC hardware accelerator — sin/cos for FOC
- FMAC — multiply-accumulate for PI filters
- Internal comparators — BEMF zero crossing detection
- 32-pin LQFP
- Programmed via STLINK-V3SET, no bootloader needed

## DSHOT Protocol Reference

| Variant | Bit rate | Frame time |
|---------|----------|------------|
| DSHOT150 | 150 kbps | ~106µs |
| DSHOT300 | 300 kbps | ~53µs |
| DSHOT600 | 600 kbps | ~26µs |

**Frame format (16 bits):**
- [15:5] = 11-bit throttle (0 = disarm, 48–2047 = throttle)
- [4] = telemetry request bit
- [3:0] = CRC (XOR of nibbles [15:12], [11:8], [7:4])

**Bit encoding:** fixed period cell, short pulse = 0, long pulse = 1

**Implementation:** DMA + timer input capture. Pre-load DMA buffer, timer triggers DMA — no CPU involvement during frame.

**Target:** DSHOT300 at 8kHz loop rate

## UART Bootloader (future — hardware consideration)

The STM32G431 system bootloader supports firmware updates over UART, enabling field upgrades without an ST-Link. Key points:

- **BOOT0 pin** (dedicated pin, not a GPIO) — pull low = boot from flash (normal), pull high = boot from system memory (bootloader)
- Hardware needs: 10kΩ pull-down on BOOT0 to GND, accessible header/test point to pull BOOT0 high, accessible NRST
- Same UART used for telemetry can be reused for bootloader — same physical wires, different mode
- Host tools: STM32CubeProgrammer (GUI) or `stm32flash` (open source CLI)
- Exact UART pin assignments for G431 bootloader to be confirmed from ST AN2606
- **Not designed yet — to be resolved during PCB hardware phase**

**Not started — ESC firmware comes first.**
