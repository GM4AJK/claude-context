# Feeltech FY6800 Signal Generator — Control Reference

**Device**: Feeltech FY6800 dual-channel DDS function generator
**USB chip**: WCH CH340 (VID 1A86, PID 7523)
**Windows driver**: https://www.wch-ic.com/downloads/CH341SER_EXE.html
**WSL2 device**: `/dev/ttyUSB0` (after usbipd-win attach)

## USB Setup (Windows + WSL2)

### One-time Windows setup
```powershell
# Install usbipd-win
winget install --interactive --exact dorssel.usbipd-win

# Bind the CH340 (do once per machine)
usbipd bind --busid 7-2        # replace 7-2 with your BUSID from 'usbipd list'
```

### Each session (PowerShell as Admin)
```powershell
usbipd list                    # confirm FY6800 shows as VID 1a86:7523
usbipd attach --wsl --busid 7-2
```

### Verify in WSL2
```bash
ls /dev/ttyUSB0    # should exist after attach
```

## Serial Protocol

- **Baud**: 115200, 8N1, no flow control
- **Format**: ASCII commands terminated with `\n`
- **Channels**: CH1 uses `M` prefix, CH2 uses `F` prefix
- **Set commands**: return no data (don't wait for response)
- **Read commands**: return ASCII value string

### Waveform types
| Code | Waveform |
|------|---------|
| 00 | Sine |
| 01 | Square |
| 02 | CMOS |
| 04 | DC |
| 05 | Triangle |
| 06 | Sawtooth (ramp up) |
| 07 | Neg Sawtooth (ramp down) |
| 11 | Sinc |
| 12 | Noise |
| 34 | ARB1 (CH1) |
| 35 | ARB2 (CH1) |
| 33 | ARB1 (CH2) |
| 34 | ARB2 (CH2) |

ARB wave codes follow the pattern: CH1 ARBn = 33+n, CH2 ARBn = 32+n (up to ARB64).
These codes are used with `WMW<NN>` / `WFW<NN>` to select an already-uploaded ARB slot.

### Command reference (CH1 — replace M with F for CH2)

| Parameter | Write | Read | Format |
|-----------|-------|------|--------|
| Waveform | `WMW<NN>` | `RMW` | 2-digit int (see table above) |
| Frequency | `WMF<14-digit-µHz>` | `RMF` | 14-digit µHz integer; read returns `XXXXXXXX.XXXXXX` Hz |
| Amplitude | `WMA<X.XXXX>` | `RMA` | Decimal V string; read returns int × 0.0001V |
| Offset | `WMO<X.XXXX>` | `RMO` | Decimal V string |
| Phase | `WMP<X.XXX>` | `RMP` | Decimal degrees |
| Duty cycle | `WMD<X.X>` | `RMD` | Decimal % |
| Output on/off | `WMN<0/1>` | `RMN` | 1=on, 0=off (reads as 1 or 255 when on) |

### Frequency encoding
```
frequency_µHz = int(hz * 1_000_000)
command = f'WMF{frequency_µHz:014d}'

Examples:
  100 Hz  → WMF00000100000000
  1 kHz   → WMF00001000000000
  10 kHz  → WMF00010000000000
  100 kHz → WMF00100000000000
  1 MHz   → WMF01000000000000
```

### Amplitude encoding
```
command = f'WMA{volts:.4f}'    # e.g. WMA1.0000 for 1V
readback: int(RMA_value) / 10000  # e.g. 10000 → 1.0000 V
```

## Arbitrary Waveform Upload

The FY6800 stores up to 64 user-defined ARB waveforms (ARB1–ARB64).
Each slot holds **8192 samples** of **14-bit** data (values 0–16383).

### Upload protocol (binary, not ASCII)

```
→  "DDS_WAVE<N>\n"        (ASCII, e.g. "DDS_WAVE1\n" for ARB1)
←  "W\n"                  (acknowledge — check before sending data)
→  16384 bytes            (2 bytes per sample, little-endian 14-bit:
                           byte0 = val & 0xFF, byte1 = (val >> 8) & 0x3F)
←  "HN\n"                 (data accepted — check for error handling)
```

Transfer time at 115200 baud: ~1.4 s for 16384 bytes.
Use `serial.Serial(..., timeout=10)` to allow sufficient read timeout.

**Cannot upload while the slot is active on a channel** — select a different
waveform first, then upload, then switch back.

### Sample value mapping

| Raw value | Output level |
|-----------|-------------|
| 0         | offset − amplitude/2  (minimum) |
| 8192      | offset                (centre) |
| 16383     | offset + amplitude/2  (maximum) |

For a digital 0 V / 3.3 V signal: amplitude = 3.3 V, offset = 1.65 V,
then `0 → 0 V` and `16383 → 3.3 V`.

### Python upload helper

```python
import serial, time

def fy_upload_arb(ser, slot, table):
    """
    Upload 8192 14-bit samples to ARB slot <slot> (1-indexed).
    ser must be open at 115200, timeout >= 10.
    table: list of 8192 ints in [0, 16383].
    """
    ser.reset_output_buffer()
    ser.reset_input_buffer()
    ser.write(f'DDS_WAVE{slot}\n'.encode())
    ser.flush()

    ack = ser.read_until(b'\n', size=16).decode(errors='replace').strip()
    if ack != 'W':
        raise RuntimeError(f'DDS_WAVE{slot}: expected "W", got {ack!r}')

    payload = bytearray(len(table) * 2)
    for i, v in enumerate(table):
        payload[i * 2]     = v & 0xFF
        payload[i * 2 + 1] = (v >> 8) & 0x3F
    ser.write(payload)
    ser.flush()

    resp = ser.read_until(b'\n', size=16).decode(errors='replace').strip()
    if resp != 'HN':
        raise RuntimeError(f'ARB{slot} upload: expected "HN", got {resp!r}')
```

### Selecting an ARB slot after upload

```python
# Select ARB1 on CH1, set frequency, amplitude, offset, enable
ser.write(b'WMW34\n')                          # ARB1 on CH1
ser.write(f'WMF{freq_uhz:014d}\n'.encode())   # frequency
ser.write(b'WMA3.3000\n')                      # 3.3 V p-p
ser.write(b'WMO1.6500\n')                      # offset → 0–3.3 V swing
ser.write(b'WMN1\n')                           # output on
```

## Python Control Class

```python
import serial, time

class FY6800:
    WAVES = {'sine':0, 'square':1, 'triangle':5, 'sawtooth':6, 'noise':12}

    def __init__(self, port='/dev/ttyUSB0'):
        self.s = serial.Serial(port, 115200, timeout=1)
        time.sleep(0.5)

    def _cmd(self, c, wait=0.15):
        self.s.write((c + '\n').encode())
        time.sleep(wait)
        return self.s.read_all().decode(errors='replace').strip()

    def _ch(self, ch):
        return 'M' if ch == 0 else 'F'

    def set_wave(self, ch, wave):
        n = self.WAVES.get(wave, wave)
        self._cmd(f'WW{self._ch(ch)}W{n:02d}')  # note: use WMW / WFW

    def set_freq(self, ch, hz):
        uhz = int(hz * 1_000_000)
        self._cmd(f'W{self._ch(ch)}F{uhz:014d}')

    def set_amp(self, ch, v):
        self._cmd(f'W{self._ch(ch)}A{v:.4f}')

    def set_offset(self, ch, v):
        self._cmd(f'W{self._ch(ch)}O{v:.4f}')

    def set_phase(self, ch, deg):
        self._cmd(f'W{self._ch(ch)}P{deg:.3f}')

    def set_duty(self, ch, pct):
        self._cmd(f'W{self._ch(ch)}D{pct:.1f}')

    def output(self, ch, on):
        self._cmd(f'W{self._ch(ch)}N{1 if on else 0}')

    def get_freq(self, ch):
        return float(self._cmd(f'R{self._ch(ch)}F'))

    def get_amp(self, ch):
        return int(self._cmd(f'R{self._ch(ch)}A')) / 10000

    def close(self):
        self.s.close()
```

### Usage example
```python
fy = FY6800('/dev/ttyUSB0')
fy.set_wave(0, 'sine')       # CH1 sine
fy.set_freq(0, 1000)         # CH1 1kHz
fy.set_amp(0, 1.0)           # CH1 1.0V p-p
fy.output(0, True)           # CH1 on
fy.close()
```

## Confirmed Working Commands (tested 2026-03-09)
- `WMW00` — set CH1 sine ✓
- `WMF00001000000000` — set CH1 1kHz ✓
- `WMA1.0000` — set CH1 1.000V ✓
- `WMN1` — CH1 output on ✓
- `RMW`, `RMF`, `RMA`, `RMN` — all read correctly ✓
- Output ON reads back as `255` (treat any non-zero as ON)

## Notes
- `fygen` Python package is NOT on PyPI — use raw `pyserial` instead (confirmed above)
- `pip install pyserial --break-system-packages` works on Ubuntu 22.04+ in WSL2
- The scope (SDS824X HD) and signal generator can be controlled simultaneously from
  the same Python script for fully automated stimulus → measure → plot workflows
- ARB waveform upload protocol reverse-engineered from
  [mattwach/fygen](https://github.com/mattwach/fygen) source (`fygen.py`,
  `wavedef.py`) — the `fygen` library itself cannot be `pip install`ed but its
  source is the authoritative reference for the binary ARB protocol and wave codes
- DSHOT300 (18750 Hz ARB playback, 512 samples/bit) is confirmed achievable;
  DSHOT600 (37500 Hz) may be marginal due to DDS quantisation at those sample rates
